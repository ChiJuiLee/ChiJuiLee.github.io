資料結構學習歷程

  • Arrays 理解記憶體連續性與索引存取優勢。
  • Linked Lists 實作單向與雙向鏈結串列，掌握指標操作。
  • Stack & Queue 應用於遞迴控制與任務排程管理。
  • Hash Tables 處理雜湊碰撞 (Collision)，理解平均 O(1) 的查找速度。
  • Binary Search Tree (BST) 學習搜尋、新增與刪除的邏輯。
  • Heaps 實作 Priority Queue，應用於排序與排程問題。
  • AVL / Red-Black Tree (進階) 理解自平衡樹如何維持搜尋效率在 O(log n)。
  • Graphs 實作 BFS (廣度優先) 與 DFS (深度優先) 遍歷算法。

核心體悟

學習資料結構後，我學會了使用 Big O Notation 來分析時間複雜度與空間複雜度。並去思考哪些資料結構會讓程式更輕鬆去完成。
